# Wallet Guard Lite Public Release Boundary

Wallet Guard Lite is a **source-closed governed execution backend/runtime** with a public API and adapter-ready integration surface.

This repository is public-safe and is intended to expose the integration, contract, deployment, and evidence layer around Wallet Guard Lite without declassifying private implementation internals.

## What this public repository includes

This repository may include public-safe materials such as:

- product boundary documentation
- API contract documentation
- request, response, reason-code, and receipt schemas
- adapter documentation
- deployment and runtime artifact documentation
- integrator guidance
- public-safe proof posture and evidence summaries

## What this public repository does not include

This repository does **not** exist to expose:

- private engine internals
- proprietary runtime composition logic
- premium heuristics
- classified harnesses
- confidential orchestration details
- private validation environments
- crown-jewel implementation internals

## Public posture

Wallet Guard Lite is currently strongest in a:

- Windows-first runtime posture
- packaged runtime artifact or controlled local runtime posture
- EVM-first adapter posture

This should not be overstated as proof of all-platform or full multi-chain parity.

## Public-safe claim discipline

The intended public-safe claim posture is:

- source-closed governed execution backend
- public API and adapter-ready integration surface
- evidence and receipt-aware design
- Windows-first strongest current runtime path
- EVM-first strongest current adapter path

The public repo should not imply:

- open-source declassification of the core
- universal consumer readiness
- complete platform parity
- complete chain parity
- final retail wallet-shell completeness

Wallet Guard Lite is intentionally compatible with third-party builder experiences.

Partners and builders may create:

- dashboards
- visual indicators
- wallet review interfaces
- treasury consoles
- extension overlays
- receipt-driven monitoring surfaces

on top of the public-safe API layer, without requiring declassification of the private engine.
