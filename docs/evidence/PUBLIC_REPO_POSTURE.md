# Public Repo Posture

## Purpose

This document defines the intended public posture of this repository.

The repository is public for evidence, validation, and credibility.

It is not public for implementation disclosure.

---

## Public-facing purpose

The public repository exists to show:

- proof that Wallet Lite works
- validation artifacts
- chain lock declarations
- public product boundary
- multi-chain execution-governance credibility

---

## Evidence-only boundary

The public repository is intentionally constrained.

Public readers should be able to see:

- what has been proven
- which chains are validated
- what the product claims
- what the product explicitly does not claim

Public readers should not receive enough implementation detail to recreate the private system.

---

## What remains public

This repository is intended to expose:

- evidence reports
- validation rollups
- lock declarations
- multi-chain declarations
- public-safe release posture
- partner-safe and investor-safe narrative material

---

## What remains private

The following are intentionally outside the public repo boundary:

- runtime implementation internals
- operational scripts
- harness logic
- private architecture depth
- private adapter logic
- internal execution paths
- monetizable implementation details

---

## Strategic rationale

This protects the moat while preserving public credibility.

The public repo proves that the system works.

It does not provide enough detail for a third party to reproduce the private engine.

---

## Final statement

This is an evidence-first public repository.

It is intentionally not a source-disclosure repository.
