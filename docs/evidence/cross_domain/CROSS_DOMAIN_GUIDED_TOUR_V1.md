## What to notice

### 1. Same primitive everywhere
Every domain is reduced to the same path:

**intent -> preview -> decision -> consequence -> receipt -> execution gate**

### 2. Different domain, same control problem
The domain changes, but the core question does not:

**Should this action be allowed to become reality?**

### 3. Same invariants recur
Across domains, the same constitutional pressures appear:

- unknown destination
- delegated authority
- threshold violation
- hidden consequence
- harm boundary

### 4. The wallet is the first proof, not the final boundary
Wallet Guard Lite is the current proof-backed implementation.
This pack shows why the architecture generalizes.

## Intended takeaway

AION / governed execution should be understood as a general execution-governance primitive.

Wallet Guard Lite is the first proof-backed surface.

This pack shows why the primitive extends across multiple consequential systems.
