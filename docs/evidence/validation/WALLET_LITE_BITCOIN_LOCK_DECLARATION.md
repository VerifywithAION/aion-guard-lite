# Wallet Lite Bitcoin Lock Declaration

## Purpose

This document declares Bitcoin as a locked Wallet Lite proof tier.

This means Bitcoin has reached a level of:

- runtime stability
- proof-backed validation
- evidence-layer representation
- deterministic preview behavior

sufficient to serve as a reference tier within Wallet Lite.

This declaration follows the same discipline used for the EVM lock.

---

## Locked baseline

The current locked Bitcoin baseline is:

- Windows-first
- Bitcoin preview runtime path
- receipt-bearing execution governance
- deterministic response surface
- Lite-safe consequence layer
- proof-backed validation artifacts
- evidence-layer surfacing

---

## What is locked

### 1. Runtime preview path

Bitcoin can:

- enter the Wallet Lite execution boundary
- return a deterministic preview response
- preserve Wallet Lite response structure
- emit receipts

---

### 2. Proof-backed validation

Bitcoin has the following validated proof families:

- BTC4 — runtime preview proof
- BTC6.3A — no-change output proof
- BTC6.3B — high-fee scenario proof
- BTC6.2 — repeat determinism proof

All of these:

- executed through the live runtime
- produced validation artifacts
- preserved receipt continuity

---

### 3. Deterministic repeat behavior

The same normalized Bitcoin request:

- produces a stable decision
- preserves reason surfaces
- preserves consequence interpretation
- preserves recommended action

This satisfies deterministic preview requirements at the Wallet Lite tier.

---

### 4. Consequence-aware preview

Bitcoin now includes a Lite-safe consequence surface capable of expressing:

- value movement
- fee posture
- output structure
- change behavior

This extends Wallet Lite from decision-only into consequence-aware preview for Bitcoin.

---

### 5. Evidence-layer representation

Bitcoin is no longer only implemented.

It is now represented in:

- validation reports
- evidence index
- proof artifacts

This means Bitcoin participates in the same proof discipline as EVM.

---

## What this means

Bitcoin is now a:

Wallet Lite locked proof tier

From this point forward:

- new chains must be compared against Bitcoin and EVM tiers
- parity claims must match these tiers
- no chain should be declared equivalent without proof

This enforces the anti-fake-parity rule.

## What is NOT being claimed

This declaration does not claim:

- full Bitcoin transaction tracing
- mempool-aware optimization
- address clustering or reputation systems
- historical wallet intelligence
- Guard Pro-level anomaly detection
- enterprise mainnet maturity across all edge cases

This is a Wallet Lite lock, not a Guard Pro lock.
