# Wallet Lite EVM Lock Declaration

## Purpose

This document declares EVM as the current locked reference tier for Wallet Lite.

This does not mean all imaginable EVM work is finished forever.

It means the critical Wallet Lite EVM baseline is now strong enough, proven enough, and coherent enough to serve as the canonical proof tier for future platform and chain expansion.

---

## Locked baseline

The current locked EVM baseline is:

- Windows-first
- EVM-first
- Sepolia-backed governed preview
- proof-pack compatible runtime path
- receipt-bearing execution governance
- consequence-aware pre-execution response surface

---

## What is locked

### 1. Core execution governance
Wallet Lite can:
- boot as a runtime
- expose a stable health surface
- return governed preview decisions
- emit receipts
- preserve deterministic decision posture

### 2. Execution proof path
Wallet Lite has:
- X5 verify OK
- C2 proof pack PASS
- Sepolia policy verify HTTP_200
- harness-backed runtime flow

### 3. Consequence-aware EVM layer
Wallet Lite now includes:
- B2 native transfer consequence awareness
- B2.2 approval proof
- B2.3 transfer proof
- richer simulation coverage introduced by B2.1

### 4. Evidence layer
These EVM capabilities are no longer only runtime behavior.
They are now represented in the proof layer through validation reports and evidence references.

---

## What this means

EVM is now the canonical Wallet Lite proof tier.

From this point onward:

- future chains should be compared against this tier
- future platform parity should be compared against this tier
- claims of parity should not be made unless they match this tier

This is the anti-fake-parity rule.

---

## What is not being claimed

This declaration does not claim:

- full trace-based execution simulation
- final post-settlement state truth
- mainnet-grade maturity across all EVM flows
- macOS parity
- Linux parity
- Bitcoin parity
- Solana parity
- Cosmos parity
- Guard Pro completion

Those remain future phases.

---

## Canonical EVM proof components

The locked EVM tier now includes at minimum:

- governed decision output
- reason surfaces
- receipts
- proof-pack compatibility
- native transfer simulation
- approval simulation

- external call surface summary

## Strategic role

This EVM lock is the foundation for the next expansion wave.

That expansion should happen in this order:

1. preserve EVM lock
2. extend future chains to the same proof grade
3. extend future platforms to the same proof grade
4. keep Guard Pro depth separate from Wallet Lite public-safe boundaries

## Final declaration

Wallet Lite EVM is now locked as the canonical hardened reference tier for the current public-safe product boundary.
