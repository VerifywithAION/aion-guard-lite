# AION Guard - Founder Brief

## One-line definition
AION Guard Lite is a multi-chain local execution-governance runtime that turns wallet activity from blind authorization into governed execution.

## The problem
Most wallets still operate under a raw authority model:

**key -> sign -> execute**

That means:
- a stolen key can become immediate loss
- risky approvals can look too similar to normal activity
- governance-sensitive actions are not structurally separated from routine actions
- the wallet stack is optimized for authorization, not disciplined execution

## The category shift
AION introduces a different model:

**intent -> preview -> classify -> govern -> execute**

This shifts wallet design from passive signing into execution governance.

## Core principle
**KINE - Key Is Not Enough**

Possession of a signing key should not automatically imply immediate unrestricted execution.

## What the product does today
The public Lite edition already demonstrates:
- local runtime execution governance
- preview-based decision posture
- structured outputs such as `EXECUTE_NOW`, `QUEUED`, `BLOCKED`, and `PAUSED`
- receipt-bearing evidence
- multi-chain support across:
- EVM
- Bitcoin hardened MVP

- Cosmos hardened MVP

## What the proof set already shows
The public repo includes evidence that:
- routine actions can remain executable
- suspicious actions escalate into review-oriented flows
- this behavior is coherent across multiple chain families
- proof reports and replay scenarios can be generated from the runtime

## Why this matters commercially
Crypto infrastructure usually forces a choice between:
- raw self-custody risk

AION creates a third path:

- with governed execution discipline

That opens room for:

- better wallet differentiation
